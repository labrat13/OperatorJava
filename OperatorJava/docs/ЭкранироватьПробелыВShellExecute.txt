Заметка создается, но не открывается, поскольку exo-open принимает путь к файлу как URL, а название содержит пробелы.
Для открытия файла через ShellExecute (командой exo-open arg1) нужно преобразовать имя файла - вместо пробелов подставить в путь %20, и впереди поставить file:
- Это есть преобразование пути файла в URI, который и требуется для exo-open.
- обычные кавычки, как в терминале виндовс, не помогают - неправильный URI с ними генерируется.

Соответственно, пути Мест тоже надо хранить в виде URI ? Пользователь должен об этом знать, чтобы правильно их указывать?
