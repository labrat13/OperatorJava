Как добавлять Процедуры и Места в БиблиотекаПроцедур.
В качестве примера использовать библиотеку GeneralProcedures.
 Она должна содержать базовые Процедуры для работы Оператора.


- БиблиотекаПроцедур должна содержать класс LibraryManager.
  - Он также служит держателем ссылок на динамические данные в памяти, используемые в Процедурах.
    - функции OnInit() и OnExit(), соответственно, для создания и удаления динамических данных.  
  - БиблиотекаПроцедур имеет Версию (LibraryManager.m_VersionString), но она сейчас не проверяется и не выводится на экран. 
  - Он передает в Движок списки Мест (LibraryManager.getLibraryPlaces()) и Процедур (LibraryManager.getLibraryProcedures()).
    В этих функциях, пока в коде, надо создать объекты Процедур и Мест, которые будут переданы в Движок Оператора.
  - Предусмотрено два типа Процедур:
    1) Процедуры, где действие выполняется при помощи командной строки, вызывающей внешнее приложение ОС.
    - поле Path объекта Procedure  должно содержать текст командной строки.
    2) Процедуры, где действие выполняется при помощи функции, реализованной внутри сборки библиотеки.
    - поле Path объекта Procedure  должно содержать путь к Процедуре как Библиотека.Класс.НазваниеМетода(аргумент1, аргумент2, аргумент3)
      - Поскольку регекс в JAVA не поддерживает русскоязычные названия групп, то имена аргументов должны бытьтолько английские.
        Да и сейчас имена аргументов не используются, их рекомендуется указывать для понимания формата функции пользователем.
- Для БиблиотекаПроцедур может оказаться полезным также добавлять свои Настройки в КоллекциюНастроекДвижка.
  Но пока неясно, как пользователь будет это использовать - настройки из БиблиотекаПроцедур нельзя изменить, они в коде прописаны.
        
- Для добавления в Библиотеку Процедур Мест, нужно создать объекты Мест внутри кода функции LibraryManager.getLibraryPlaces(), и установить их свойства согласно примеру там же. 
  - файловые пути Мест часто привязаны к имени аккаунта пользователя, поэтому Места внутри БиблиотекаПроцедур следует создавать с учетом этих обстоятельств.
  - Вообще, Места, используемые пользователем, следует создавать в БД Оператор, там их проще редактировать, чем внутри кода БиблиотекаПроцедур.  
  - в БиблиотекаПроцедур следует добавлять стабильные Места, которые не зависят от версии и языка Linux, названия аккаунта пользователя и погодных условий.  

- Для добавления в Библиотеку Процедур Процедуры командной строки нужно создать объект Процедуры внутри кода функции LibraryManager.getLibraryProcedures() и установить ее свойства согласно примеру там же.
 
- Для добавления в Библиотеку Процедур Процедуры с вызываемой функцией нужно:
  1. Создать класс и пометить его аннотацией, в соответствии с примером.
  - Если подходящий класс уже был создан ранее, просто перейдите к пункту 2.
  - Предполагается, что БиблиотекаПроцедур и ее одноименный java package служит семантической группой, описывающей некую предметную область для операций. Например, математические операции, или операции инвентаризации предметов.
  - Предполагается, что класс служит семантической группой, категорией для схожих операций.
  2. В выбранном классе создать статическую функцию с аннотацией, в соответствии с примером.
  3. Добавьте в созданную функцию класса код, соответствующий требуемой работе, в соответствии с примером.
  4. измените аннотацию функции класса в соответствии с памяткой.
