---------------------
распределение: Предварительная версия.
/home/jsmith - user folder как база
Название приложения: Operator
Папка приложения: /home/jsmith + .Operator 
Файл настроек приложения: .Operator/settings.ext
Файл БД приложения: .Operator/db.sqlite
Папка сборок команд: .Operator/proc
Папка приложений-процедур: .Operator/apps
---------------------
Путями и файловыми операциями управляет класс FileSystemManager
Корневая папка Оператор должна начинаться с точки - тогда она будет скрыта, чтобы она не мешала пользователю в домашней папке.
 Чтобы он там ничего не испортил, случайно забравшись не туда.
Файл настроек приложения и файл БД размещаются в каталоге рядом с исполняемым файлом приложения.
Для сборок процедур выделена папка .Operator/proс, в ней сборки процедур размещаются в своих подпапках.
 Каждая папка сборки процедур должна иметь стандартную структуру:
 - todo: уточнить в последующих итерациях.
 - файл описания сборки процедур - для Оператор и пользователя, должен содержать версию сборки процедур.
    Может одновременно быть файлом настроек для данной сборки процедур.
 - файл сборки процедур
 - подкаталог apps для приложений, используемых в качестве процедур.
Для приложений, используемых в качестве процедур, выделена папка Operator/apps, в ней приложения-процедуры размещены либо в подпапках, либо внавал.
------------------------
Схема каталога:
- home/jsmith				- Домашний каталог пользователя
  - .Operator				- Каталог Оператор
    - settings.txt			- Файл настроек приложения
    - db.sqlite				- Файл БД приложения
    - apps/					- Папка приложений, вызываемых в качестве процедур.
      - run-cmd.cmd			- Файл приложения-процедуры, вызываемый при исполнении команды.
    - proc/					- Папка сборок команд
	  - assembly1/			- Папка сборки assembly1
	    - description.xml	- Файл описания сборки процедур
	    - assembly1.jar  	- Файл сборки процедур
	    - apps/				- Подкаталог для приложений, используемых в качестве процедур.
	      - run-cmd.cmd		- Файл приложения-процедуры, вызываемый при исполнении команды.
